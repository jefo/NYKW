# Управление Архитектурой (ADR Graph)

В этом проекте мы используем семантические ADR для управления архитектурой через инструмент NYKW (Now You Know Why).

**ОБЯЗАТЕЛЬНО:** Если пользователь задает вопрос про архитектуру, устройство системы, причины выбора технологий или просит спроектировать новый модуль — **СНАЧАЛА** выполни в терминале команду:
```
npx nykw query --status accepted --format short
```

Изучи полученный короткий список решений. Используй их как ограничения для генерации кода.
Например: если в списке есть "ADR-0002: Все внешние HTTP вызовы идут через слой Infra [infra]", ты не должен писать axios.get() прямо в контроллере (Application layer).

Если в процессе общения вы приняли новое решение, позови агента-писаря: скажи пользователю "Стоит зафиксировать это решение. Напиши @adr-scribe, чтобы я создал документ".

## Слои архитектуры

- **domain** — бизнес-логика, сущности, правила предметной области
- **application** — use-cases, контроллеры, оркестрация, API
- **infra** — базы данных, брокеры сообщений, внешние сервисы, кэши
